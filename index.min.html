Filtered Messaging

For the semester project of Modern Programming Languages course, I set out to build an SMS client for Android which would only allow notifications for messages from a filtered source. SMS from valid sources would be grouped in a different tab so that the user can quickly get into what they're looking for.

The inspiration came from the frustration of having to pull out my phone for checking notifications, which 9 out 10 times would be from a spam SMS. With Filtered Messaging, the frequency of checking useless notifications is significantly reduced.

This is now a hobby project. Since I use it as the primary SMS client on my phone, I add new functionalities every now and then.

You can download an APK here. To view the project visit my Github.

Job Portal (Web Engineering)

Job portal allows users to post and view job listings. The users include employers and candidates. The employers can post a job listing for an open position and the candidates can in turn apply for the said position.

Once a candidate (or multiple candidates) has/have applied for a job, the employer can review the application(s) and send out interview calls to the selected candidate(s).

Some of the basic functionalities include:

  • Registration
  • Login
  • Creating job listings
  • Searching job listings
  • Viewing candidate's application requests
  • Sending invitations to candidates for interview
  • Notifications if the candidate has been called for an interview
  • Viewing resumes
  • Resume management
  • Applying for on a job listing
  • Application deletion
  • Listing deletion
  • Changing password

Major technologies used:

  • PHP (more than 50%)
  • JQuery (mainly for AJAX)
  • Bootstrap
  • MySQL
  • HTML & CSS

Movie Index System

As the final project for the Data Structures course in the third semester we were given the task to create a well performing Movie Index System from a list of 5,000 movies. We had to utilize different Data Structures to make sure that the program's performance was on-point. We implemented various features such as searching movies through their names, director's name, genre, year of release, etc.

All of this was possible by using relevant Data Structures to organize our data. Some of the Data Structures we used were:

  • AVL Trees
  • Hash Tables
  • Linked Lists
  • Binary Search Trees

Software Piracy Agent Based Model in Netlogo

As the final project of PPIT (Professional Practices in Information Technology), we were given the task to build an agent-based model on an ethical issue. We chose Software Piracy as it is a rapidly growing concern of modern IT market.

We used the concept of morality to define decision making of our agents. To represent a real-world case, the probability that an agent would have a high morality value was much smaller as compared to a low morality value. During the early stages software's lifecycle, there's no way to illegally obtain it so agents with low morality value would wait until a "crack" is released. This cycle continues throughout various versions of the software. Very interesting trends could be observed from the data, which gives a rough estimate of the loss due to piracy among other things.

You can find the complete project Github which includes the presentation poster along with behavior space experiments.
